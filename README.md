# toosmart

## Markdown → HTML пайплайн

Статичные статьи собираются скриптом `node build.js`. Он читает исходные `.md` файлы из `source/articles`, прогоняет их через `marked`, санитизирует HTML через DOMPurify и сохраняет итог в `public/articles`.

### Зачем нужен `source/config.json`

Конфигурация описывает порядок статей, локализованные заголовки и связи для кнопки «Далее». Если файл отсутствует, скрипт определяет статьи автоматически по алфавиту. Структура файла:

```json
{
  "articles": [
    {
      "id": "intro",
      "title": "Введение в курс",
      "markdown": "intro.md",
      "next": "basics"
    }
  ]
}
```

Дополнительные поля не запрещены: они проходят дальше к шаблону и могут использоваться для расширенной навигации.
